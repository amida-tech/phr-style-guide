4.0 - Navigation

Covers structure and styling of navigation elements throughout the site.

4.1 - Primary Navigation

A primary navigaton bar across the top of the page provides a static placeholder for information, while freeing up the body of the site for the display of content specific information.

In applications which have multiple users, and in which a user may be viewing records of various individuals, it is important to differentiate between the current user and the record of the individual. A common design practice is to embed the current user's information in the navigation bar. This provides user-level context and will allow a system user to visually differentiate between themselves, and the individual associated with any given record.

If search is a feature of the site, it should be incorporated into the navigation bar. This will allow easy access from any area of the site.

4.2 - Profile

Particularly when viewing medical records, it is important to have context as to the person associated with the medical record which is being viewed. By appropriately incorporating a profile 'snapshot' for identification, users can easily differentiate record associations on a case by case basis. This component should be generally embedded in the top left corner of appropriate pages where identification is necessary, taking priority over any secondary navigational components.

4.3 - Secondary Navigation

Secondary navigation should be used on sub-pages to navigate between various components contained within that area. For instance, a sub navigation could be used to navigate through various parts of a medical record.

If a sub-page requires profile information to be displayed, the navigation should appear below the profile area.

Default styling

 

<div class="btn-group-vertical">
  <button type="button" class="btn btn-default">Allergies</button>
  <button type="button" class="btn btn-default">Medications</button>
  <button type="button" class="btn btn-default">Lab Results</button>
</div>

4.4 - Breadcrumbs

Breadcrumbs should be included for sections of the site in which a user must drill down into data. This is necessary in order to provide a clear method for the user to orient themselves and walk backwards if needed. The breadcrumb is secondary content, and should not be contained within a body panel, but rather set against the background directly; it should use the entire width of the page. Breadcrumb links should start a navigational layer below links available from the navigation bar to minimize redundancy.

Default styling

My Record > Allergies > Review Updates

 

<div>
  <div class="breadcrumb"><a>My Record</a> &gt; <a>Allergies</a> &gt; <a>Review Updates</a></div>
</div>
