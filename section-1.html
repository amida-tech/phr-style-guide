1.0 - Layout

General layout parameters of the site.

1.1 - Responsive Design.

Supporting multiple device form factors is critical for the display of health records. For instance, records are frequently viewed on mobile phones during encouters with medical providers.

To gracefully degrade, this style guide suggests the use of the responsive grid system in Twitter Bootstrap. This system uses media query break points to trigger degradation across devices of nearly any form factor. More information on the responsive grid system can be found at getbootstrap.com.

As a general rule, care must be taken to ensure there isn't a loss of information during transition between form factors. Details about a record component, for example an allergy's reactions, if not immediately viewable following transition must still be accessible. This means clear visual indicators for a user to access collapsed or hidden data.

Note how by adjusting the browser window, the below blocks dynamically stack.

Default styling

 

<div class="col-md-12">
  <div class="col-md-6"><button type="button" class="btn btn-primary btn-block">Block One</button></div>
  <div class="col-md-6"><button type="button" class="btn btn-default btn-block">Block Two</button></div>
</div>

1.2 - Page Structure

The primary navigation of the site should be statically fixed to the top left of the page. The navigation should feature a user picture and name, so at any time there can be certainty as to who's record is being viewed.

Default styling

 

<div class="btn-group-vertical">
  <button type="button" class="btn btn-default">Home</button>
  <button type="button" class="btn btn-default">My Record</button>
  <button type="button" class="btn btn-default">My Files</button>
</div>

1.3 - Footer Usage

The footer of the site can be modified further to include additional navigation links such as 'About Us', 'FAQ', and 'Contact Us'. Who and what these links point to depend on the context of the PHR.
